# Роли агентов

## Сбор логов

Вход: `raw_logs`, `source_ip`, `host`.

Выход: нормализованные события с полями `message`, `severity`, `source_ip`, `host`.

Бизнес-правила: ошибки входа получают уровень `warning`, признаки вредоносной активности получают `critical`.

## Корреляция событий

Вход: массив нормализованных событий.

Выход: число неудачных входов, число критических событий, группировка по IP.

Бизнес-правила: несколько неудачных входов от одного источника повышают риск.

## Детекция атак

Вход: результат корреляции.

Выход: `attack_detected`, `risk_score`, `attack_type`, `reasons`.

Бизнес-правила: 3 и более неудачных входа дают признак bruteforce; критическое событие повышает риск до high-risk intrusion.

## Блокировка трафика

Вход: результат детекции и IP-источник.

Выход: итоговое решение о блокировке.

Бизнес-правила: если атака обнаружена, IP добавляется в Redis-множество `siem:blocked_ips`, увеличивается счётчик блокировок.
